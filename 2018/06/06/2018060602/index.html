写在前面

现在我们来学习新的数据结构：链表，这个和之前介绍的3种数据结构：动态数组，栈，队列是不同的，至于那里不同，我们现在就来学习吧。

简单回顾

我们在前面介绍的3种数据结构：动态数组，栈和队列，它们虽说不同，但底层都是依托静态数组而实现的，而且都是靠resize来解决容量问题。链表则不同，它是一种真正的动态数据结构。因此，这也是我们到目前为止真正接触到的第一个动态的数据结构。

链表的重要性

链表是重点，也是难点。它是最简单动态数据结构，后面我们还会学习更多的动态数据结构，例如二分搜索树，平衡二叉树，红黑树等。那些动态数据结构都可以在理解链表的基础上进行学习。

同时链表可以让我们加深入对引用(也就是C++中的指针)和内存管理的认识，以及对于更深入的理解递归有极大的好处。还有链表可以辅助组成其他的数据结构。

链表LinkedList

链表是一种数据存储在“节点”(Node)中的动态数据结构，里面不仅存放了元素本身，还存放了指向一下元素的指针。

Code

你可以将其类比到火车车厢，每节火车不仅可以载人，还可以和其他火车进行连接。当然最后一节车厢就没有连接，就是空的。是不是很形象，的确数据结构很多就是来源于生活。

很明显链表的优点就是，实现了真正的动态，不需要处理固定容量这个问题。这个不像数组，你一下子必须开辟出一片空间，同时还需要考虑空间不够用或浪费问题。而链表就不需要了，它是你需要多少个数据，就生成多少个节点，并将其进行挂接，从而实现了动态性。

但是链表也有一个非常明显的缺点，就是不支持随机访问。不能像数组一样，给定一个索引就能立即找到对应的元素。在底层中，数组开辟的空间在内存中是连续分布的（也就是在内存中连续开辟内存），我们完全可以直接寻找索引对应的偏移，然后计算出数据所存储的内存地址，使用O(1)复杂度就能找到该元素。而链表各个元素之间依靠next连接，每个节点存储地址不同，只能通过next顺藤摸瓜来找到我们想要的元素。因此链表这种数据结构，对于查询操作来说是不友好的，特别是当你要查找末尾元素的时候。

我们前面说过数组最好适用于索引有语义的场景，但是身份证就不能使用数组了，因为这需要开辟很大的内存空间。（这也就是我们为什么需要创建动态数组的目的，因为那个时候索引就是没有语义的场景），因此数组最大的优点就是支持快速查询。

现在我们思考在索引没有语义的情况下，有没有不通过创建动态数组来实现同样的目的呢？答案就是可以使用链表，它的优点就是动态。

后面我们会介绍在不同的业务场景下如何选择性的使用数组和链表以及其他的数据结构。说了这么多，接下来我们就来看看链表的实现吧。

链表的时间复杂度分析

对于添加操作而言，因为往链表尾部添加需要遍历整个链表节点，因此O(n/2)可以看做是在操作中间的节点。那么总的来说添加操作的时间复杂度就是O(n)。

对于删除操作而言，其实和添加操作是非常相似的，时间复杂度也是O(n)：

对于修改操作，大家注意，因为链表不支持随机访问，因此时间复杂度是O(n)。

对于查找来说，时间复杂度是O(n)，这里没有之前所说的find方法，因为find返回的是索引号，而链表不支持随机访问，因此这个方法就没有意义了。

通过上面的观察，我们发现链表的增删改查全都是O(n)级别的，比数组看起来差很多，而且链表没有索引，无法像数组一样支持随机访问，那么我们还研究它干嘛呢？
下面的图片说明了，如果增加和删除只是对链表头进行操作，那么它的性能还是不错的，或者说只是查询链表头，那么它们的时间复杂度是O(1)。当然了，如果我们对链表进行改进，那将会比数组节省一定的空间，而且链表是最基础的动态数据结构，它还能对二叉树，平衡二叉树的学习起到辅助作用，因此我们还是需要花时间去研究它的。

使用链表实现队列

我们又来看这张图片，现在我们想利用链表来实现队的功能，这可以吗？答案是可以的，但是我们不能直接使用我们的链表。因为链表只有在开头进行添加，删除和查询是O(1)的复杂度，而队列是两端都要进行操作的，因此势必会出现一端为O(1），一端为O(n)的情况。在使用数组时我们也遇到了这个问题，因此我们产生了使用循环队列的方式，那么这里我们是不是也可以借鉴这种思想呢，对，我们差不多也是这样。

现在我们思考一下，为什么在链表中我们对于链表头部的操作都是非常简单的呢？因为我们有一个标识head，由于它的存在，我们就非常清楚操作的元素。同样如果我们的尾部也存在这样的一个标识tail，那样不就也是很简单的么？对，通过设置一个tail标识，使得从两端插入元素这一操作变得很容易实现。

但是对于删除操作而言，尽管有tail的存在，我们依旧还是无法进行简单操作。因为
tail它前面的那个节点是不容易找到的，我们必须得遍历一遍才能知道。通过这样的分析，我们就知道了： head的存在使得添加和删除操作都很容易实现，但是tail的存在只是让添加操作变得容易，删除操作依旧还是困难。

既然这样的话，我们让tail执行添加操作，head执行删除操作。也就是head为队首，负责出队；tail为队尾，负责入队。注意一下，此时是没有dummyHead的，一定要注意链表为空的情况。（因为我们这是在队列开头和结尾进行操作，不涉及中间，因此就没有必要设置虚拟头部了）
