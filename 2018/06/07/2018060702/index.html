链表与递归

我们知道递归是组建逻辑非常重要的一部分，很多人以为递归只能编写抽象逻辑，其实不然。

可视化算法中随机生成迷宫算法，分形图的绘制都与递归密不可分；推箱子的自动求解也是一个递归的过程，高级排序算法也会使用到递归，因此递归非常重要。

说了那么多，那么什么才是递归呢？
递归的本质上，是将原来的问题，转化为更小的同一问题（小到不能再小为止）。

链表的天然递归性质

第一个链表，我们可以理解为是0这个节点，后面再挂了一个更短的链表，最后挂接了NULL。我们知道，在第一个链表里面，除去0这个节点以后，它本身是一个新的更短的链表，只不过比原始链表少了一个节点0。但是对于后面更短的链表来说，这个节点1却变成了它的头结点。以此类推，空链表也是一个链表，它是最基本的链表。

递归调用的调试

你可以使用打印输出法或者是debug单步跟踪法，你如果之前没看过这个文章，建议使用debug单步跟踪法，因为这个就是IDE自带的方法。

不过在此之前，我们需要介绍一个概念，或许这不是概念，因为在前面的调试介绍中你就已经不知不觉的发现了，只不过不知道这个术语，又或者说没太注意这个。那是什么呢？对，递归深度。说白了就是每次调用时，你需要进入下一个函数，此时递归深度就会加一，这样说你肯定就恍然大悟了。

Code

我们从上面的运行结果中可以看到：递归深度在不断增加，不断在更短的链表中来删除链表，直到满足最基本条件：在空链表中删除元素6，结果仍然为空，开始return为止，然后一直往回执行。

当递归深度变短时，程序执行就会回到上一层，这样不断返回。得到上一层返回后，只需要判断该递归深度中，当前的头结点是否需要删除即可。

尽管我们这里的最终解决是使用了四行代码，但是这背后的功夫绝不止四行，都是不断的进步和改进来的，因此想要真的学习算法其实真的很不容易。
