Go语言学习笔记（1）：概述

概述

Go语言的设计初衷是针对其他语言的痛点进行设计，并加入了并发编程，可以说是为大数据，微服务和并发而生的通用编程语言。它的开发者是C语言的开发者，Go是一门静态语言。其实目前Go大多都用在项目的转型上面，是许多项目转型的首选语言。Go语言非常特别，和你目前所知的Java、Python等不同，它没有“对象”，没有继承和多态，没有泛型，也没有try/catch。我猜想此时如果你是JavaScrpit、Java或者是C++的语言使用者，你可能就觉得非常难以接受了。不过没关系，Go语言尽管没有上面的那些，但是它具有接口，函数式编程和CSP并发模型(goroutine+channel)，有这些就够了。Go语言其实非常好学，因为语法简单。这套笔记是我参考很多书籍和资料整理的，因此会包括Go的基础语法（这些语法可以解决实际中的问题），综合运用（包含一些经典的算法、例题和微型的项目）以及实战内容（搭建一套结构复杂的分布式爬虫系统），当然由于本笔记记录的时间较长，此时我可能会使用Go搭建一些网站，会在适当时候释放源代码，供大家交流学习。我是一个比较喜欢折腾的人，所以可能会在一些场景中将Go语言和其他语言（Java、Python）进行对比，然后发现Go语言的一些魅力，最重要的是了解Go语言设计的哲学思想。其实通过学习Go，会让你对现掌握的语言有不同的理解，我想这就是所谓的触类旁通吧。不过我个人建议你在学习Go语言之前能忘掉现有的语言，无论是代码规范，还是语法，甚至是设计思想。

Go语言包含的内容

这里所说Go语言包含的内容这种说法其实并不准确，应该是目前我所学习的Go语言知识，所以可能罗列的不是很全，但我尽量做到全。a）基本语法：变量、选择、循环、指针、数组、容器等；b）面向接口：结构体、duck typing、组合思想等；c）函数式编程：闭包以及一些多样的例题；d）工程化：资源管理，错误处理，测试和文档、性能调优等；e）并发编程：goroutine、channel、调度器以及些多样的例题；f）完全使用Go语言原生开发一套比较简单但架构复杂的分布式爬虫系统。

Go语言的安装

点击这里Go官网，获取Go语言的官方网址，选择合适自己的版本，如果官网打不开，则点击Go语言中文网，进行下载。

如果你是小白又不想折腾，则按照默认方式直接下一步安装即可。

如果你想折腾，自定义Go语言的安装位置（假定安装路径为G:\Applications\Go），同时想把自己的Go项目放在I:\Go\GoTest目录，则在系统变量中添加以下信息：

变量名称Path，变量值G:\Applications\Go\bin;
变量名称GOPATH，变量值I:\Go\GoTest;
变量名称GOROOT，变量值G:\Applications\Go;

其实也就是说Go的全局环境是Path；GOPATH是代码中引用的包所在的位置，可以看成是工程目录，可以设置多个（或者说是项目的存放地址）；GOROOT是Go语言的安装目录，类似于JAVA中的JAVA_HOME，之后保存设置即可。

安装后在dos环境下输入go version，若出现诸如此类的版本信息表示安装成功go version go1.12.2 windows/amd64，可参考这篇文章golang第二课—go语言介绍及版本查看。在Go语言开发过程中，个人推荐使用的IDE是GoLand，因为它功能比较强大，上手容易，特别是使用过IDEA，PyCharm的童鞋就更友好了。再推荐两个学习Go语言的网站:Go语言中文网和Golang中国社区，至此Go语言的安装就完成了。

Code

package main定义了包名。你必须在源文件中非注释的第一行指明这个文件属于哪个包，如：package main。package main表示一个可独立执行的程序，每个 Go 应用程序都包含一个名为main的包。

import "fmt"告诉 Go 编译器这个程序需要使用 fmt 包（的函数，或其他元素），fmt 包实现了格式化 IO（输入/输出）的函数，类似于Java的System.out包，不过Java中的lan包是会自动导入，但是Go不一样，需要手动导入。

func main(){}是程序开始执行的函数。main 函数是每一个可执行程序所必须包含的，一般来说都是在启动后第一个执行的函数（当然如果有init()函数则会先执行该函数），和Java中的public static void main(String [] args){}非常相似。

/* 第一个Go语言程序 */是注释，在程序执行时将被忽略。以 // 开头的是单行注释， /* 开头，并以 */ 结尾，且不可以嵌套使用的是多行注释。多行注释一般用于包的文档描述或注释成块的代码片段。

fmt.Println(...)可以将字符串输出到控制台，并在最后自动增加换行字符 \n。 这个和使用 fmt.Print(“hello, world\n”) 得到的结果是相同的。 Print 和 Println 这两个函数也支持使用变量，如：fmt.Println(arr)。如果没有特别指定，它们会以默认格式将变量 arr 输出到控制台。这个和Java中的print、println方法非常相似。

关于包和文件，需要注意以下3点：
1、文件名与包名没有直接关系，不一定要将文件名与包名设置相同的名称。
2、文件夹名与包名没有直接关系，并非需要一致。
3、同一个文件夹下的文件只能有一个包名，否则引发编译错误。
